# Changelog

本文档记录青卷在 `2026-03-25` 前后完成的主要功能更新与行为变更。

## [Unreleased] - 2026-03-25

### 清理

- 漫画翻译链路已从“外部 AI 命令执行”切换为后端内置图片编辑流程，不再维护命令模板、工作目录与超时配置。
- 删除设置页中的漫画 AI 命令相关输入项；漫画译图现在直接复用默认翻译提供商配置。
- 删除设置页中的 LaMa 相关选项、后端 `/settings/lama/install-deps` 接口，以及对应的前后端类型与默认配置。
- 清理 sidecar 构建脚本、PyInstaller spec 与 `python-backend/requirements.txt` 中仅本地漫画修复链路使用的打包项与依赖。
- 删除未再使用的内置中文字体资源与许可证文件，并移除对应历史描述。
- 漫画任务日志保留并继续工作，内置图片编辑请求过程会持续写入日志系统。

### 新增

- 新增漫画工作流，支持将远程链接识别为 `漫画` 类型，并进入按话 / 按页的导入、下载与阅读流程。
- 新增 `18comic.vip` 与 `bikawebapp.com` 漫画站点支持，覆盖预览、章节抓取、分页图片下载与本地阅读。
- 新增 `Grok2API` 翻译提供商配置入口，支持作为默认翻译提供商使用。
- 新增 Bika 账号自动创建与登录保存逻辑；用户不手填账号时，首次抓取可自动注册并复用本地凭证。
- 新增漫画翻译图片渲染版本标记，用于识别旧版缓存图片是否仍可继续复用。

### 改进

- 阅读进度从“只记录章节”升级为“章节 + 章节内位置”持久化。
- “继续阅读”现在会尝试恢复到上次退出时的段落 / 图片附近，而不是只打开对应章节。
- 切换到上一章 / 下一章时，会自动回到页面顶部开始阅读。
- 切换原文 / 译文阅读模式时，尽量保持当前阅读位置不丢失。
- 已安装桌面版默认将数据库、书库、翻译结果与资源写入主程序同级 `data/` 目录。
- 安装目录下 `data/` 为空时，会自动尝试迁移旧版 `%LOCALAPPDATA%/QingJuan/data/` 的历史数据。
- README 已同步更新为“小说 + 漫画”双工作流说明，并补充当前真实的数据目录行为。

### 修复

- 修复 18comic 阅读阶段的章节图片修复链路，降低图片乱序 / 切片异常导致的阅读失败概率。
- 修复“继续阅读”虽然能显示百分比、但恢复时机过早导致无法跳到真实位置的问题。
- 修复桌面端非阅读页侧栏跟随页面一起滚动的问题。
- 修复侧栏收缩后 Logo 与折叠按钮发生重叠的 UI 问题。
- 修复漫画译文图片在极端长文本下可能超出原始文字框的问题。
- 修复升级到新版渲染器后仍继续显示旧版 `translated.png` 缓存、导致阅读器长期显示旧“问号图”的问题；旧缓存现在会自动退回为“原图 + 本页译文”。
